## Guide Characteristics

All user guides follow these standards:

### Structure

1. **User Story**: Real-world scenario explaining why and when to use feature
2. **Step-by-Step Instructions**: Every button, field, dropdown, with exact locations
3. **Specific Values**: Concrete examples using TechGadgets project (never generic placeholders)
4. **Visual Descriptions**: Detailed UI element descriptions (colors, positions, sizes)
5. **Options Explained**: Every dropdown option with when to use it
6. **Why These Choices**: Justification for each configuration decision
7. **Verification Steps**: How to confirm each step worked correctly
8. **Troubleshooting**: Common issues and solutions
9. **Best Practices**: Tips and recommendations
10. **Checklists**: Summary checklist before moving to next section

### Detail Level

- **Field-by-field**: Every input field explained (label, type, placeholder, validation)
- **Button-by-button**: Every action button with exact location and outcome
- **Click-by-click**: Every mouse action and resulting page transition
- **Character counts**: Text field limits and current usage shown
- **Processing times**: Expected duration for operations
- **Visual feedback**: Loading states, success animations, error messages

### Example Project Consistency

All guides use the **TechGadgets Inc.** example:

- Electronics retailer (smartphones, laptops, tablets, accessories)
- TechGadgets Support Bot (main assistant agent)
- 25 product documents (manuals, policies, FAQs)
- Team members: Sarah Chen, Mike Rodriguez, Lisa Thompson
- Specific products: UltraBook 15, TabPro 10, XPhone Pro, etc.

### Code & Configuration

- **Actual code**: Real configuration JSON, system prompts, API requests
- **No pseudocode**: Everything is production-ready, copy-pasteable
- **Complete examples**: Full responses, not truncated with "..."
- **Real metrics**: Actual token counts, costs, processing times

### Length & Comprehensiveness

- **Target**: 700-1,000+ lines per guide (matching assistant agent guide length)
- **No shortcuts**: Every feature, every option, every edge case covered
- **Progressive detail**: Builds from simple to advanced within each guide
- **Cross-references**: Links to related guides for additional context
